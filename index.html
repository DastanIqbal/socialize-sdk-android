IMPORTANT NOTE FOR v0.2.x Users

As of v0.3.0, the Socialize SDK for Android no longer requires any external libraries or resources :)

If you have a previous installation of the Socialize SDK for Android, you should remove any 3rd party libraries that were part of the previous release.  These are now included the single socialize-x.x.x.jar file, as are the external configurations previously required for Android 2.1 builds.

Installing the SDK

The Socialize SDK is delivered as a single JAR file, simply copy the socialize-x.x.x.jar file from the dist folder to the libs path of your Android project.

Applications targeting older versions of Android (1.6 and below)

Socialize does not support Android versions below v2.1-update1

Configure your AndroidManifest.xml

Add the following lines to your AndroidManifest.xml under the <manifest...> element

<manifest...>

<application.../>

<uses-permission android:name="android.permission.INTERNET"/>

<uses-permission android:name="android.permission.READ_PHONE_STATE" />

</manifest>

Initializing the Socialize Instance in your application

The Socialize SDK should be initialized in the onCreate() method of your Activity, and destroyed in the onDestroy() method:

import com.socialize.Socialize;

protected void onCreate(Bundle savedInstanceState) {

        super.onCreate(savedInstanceState);

        

        Socialize.init(this);

        ...

}

protected void onDestroy() {

        Socialize.destroy(this);

...

        super.onDestroy();

}

Calling the Socialize API

Every call against the Socialize SDK MUST be authenticated.  On the first successful call to “authenticate” the credentials are automatically cached in the application so subsequent calls are much faster.

ALL calls to the Socialize SDK are asynchronous.  This means they will not interrupt your users’ use of your application.  You are notified of the outcome of calls to the Socialize service via a SocializeListener passed into each call to the Socialize SDK.

Authentication

To authenticate your application for Socialize, simply call the authenticate method:

Socialize.getSocialize().authenticate(

consumerKey,

consumerSecret,

new SocializeAuthListener() {

        public void onAuthSuccess(SocializeSession session) {

                // Success!

        }

        public void onAuthFail(SocializeException error) {

                // Handle auth fail

        }

        public void onError(SocializeException error) {

                // Handle error

}

});

The consumerKey and consumerSecret values are provided to you when you registered at the http://getsocialize.com website

Once your application is authenticated, you can skip the call the authenticate for subsequent uses of the SDK and simply check the isAuthenticated() state:

if(Socialize.getSocialize().isAuthenticated()) {

        // Ready to go!

}

We recommend however that you just use the authenticate method when ever you want to interact with the Socialize service.  This will ensure you are always authenticated and does not impose any significant overhead.

Calling SDK methods

To make a call to the Socialize SDK, simply use one of the methods exposed on the top level Socialize class:

// Make sure we are authenticated

if(Socialize.getSocialize().isAuthenticated()) {

        // Post a like

Socialize.getSocialize().like("http://someurl.com",

new LikeAddListener() {

        public void onCreate(Like entity) {

                        // The like was created!

        }

        public void onError(SocializeException error) {

                        // Oops! Something went wrong!

        }

});

}

Alternate (Recommended) approach with authentication

Socialize.getSocialize().authenticate(

        consumerKey,

        consumerSecret,

        new SocializeAuthListener() {

                public void onAuthSuccess(SocializeSession session) {

                        // Post a like

                        Socialize.getSocialize().like(

                        "http://someurl.com",

                        new LikeAddListener() {

                                public void onCreate(Like entity) {

                                        // The like was created!

                                }

                                public void onError(SocializeException error) {

                                        // Oops! Something went wrong!

                                }

                        });

                }

                public void onError(SocializeException error) {

                        // Handle error on auth

                }

                public void onAuthFail(SocializeException error) {

                        // Handle authentication failure

                }

        });

Socialize UI

Pre-built Views for Socialize

As of v0.4.0 of the Socialize SDK we are providing pre-built UI views that can quickly and easily be dropped in to your app, saving you the time of building these views yourself!

v0.4.0 introduces the “Comment View” which provides the creation and viewing of comments associated with an entity (URL).  Currently this supports only anonymous users, but future releases will incorporate additional authentication systems such as Facebook and Twitter.

Installing the SDK for with Socialize UI

The installation process is the same for the Socialize UI as with the standard SDK.

The Socialize SDK is delivered as a single JAR file, simply copy the socialize-x.x.x.jar file from the dist folder to the libs path of your Android project.

Configure your AndroidManifest.xml

Add the following lines to your AndroidManifest.xml under the <application...> element

<manifest...>

<application...>

<activity android:name="com.socialize.ui.comment.CommentActivity"/>

</application>

</manifest>

Add the following lines to your AndroidManifest.xml under the <manifest...> element

<manifest...>

<application.../>

<uses-permission android:name="android.permission.INTERNET"/>

<uses-permission android:name="android.permission.READ_PHONE_STATE" />

<!-- Optionally add ONE of the following to include location data in comments -->

<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>

<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>

</manifest>

Adding comments to your app using the Socialize Comment View

To incorporate the comment view into your app you simply need to use the new SocializeUI class.

NOTE: If you are using Socialize UI exclusively (that is, you’re not calling the SDK directly) you do not need to initialize the Socialize SDK yourself.  This is done for you in the UI components themselves.

First you need to specify your Socialize app key.  This can either be done in code:

import com.socialize.ui.SocializeUI;

...

SocializeUI.getInstance().setSocializeCredentials(

consumerKey,consumerSecret);

Or by creating a configuration file in your assets path called socialize.properties

Example socialize.properties file located in the “assets” path of your project

# Socialize App Key

socialize.consumer.key=00000000-0000-0000-000000000000

socialize.consumer.secret=00000000-0000-0000-000000000000

When you want to launch the comment view, simply call showCommentView from the SocializeUI instance:

import com.socialize.ui.SocializeUI;

...

SocializeUI.getInstance().showCommentView(SampleActivity.this, “http://someurl”);

Here’s an example of calling it on a button click:

(replace “http://someurl.com” with your entity key)

Button btn = (Button) findViewById(R.id.btnCommentView);

                

btn.setOnClickListener(new OnClickListener() {

public void onClick(View v) {

        String entityKey = “http://someurl.com”;

        SocializeUI.getInstance().showCommentView(SampleActivity.this, entityKey);

        }

});

Here’s a complete example in an Activity:

import android.app.Activity;

import android.os.Bundle;

import android.view.View;

import android.view.View.OnClickListener;

import android.widget.Button;

import com.socialize.ui.SocializeUI;

public class SampleActivity extends Activity {

        public void onCreate(Bundle savedInstanceState) {

                super.onCreate(savedInstanceState);

                

                setContentView(R.layout.main);

                

                // These can be specified in a config file

                // called socialize.properties and places in your assets path.

                final String consumerKey = "00000000-0000-0000-000000000000";

                final String consumerSecret = "00000000-0000-0000-000000000000";

                

                // Your entity key.  May be passed as a Bundle parameter to your activity

                final String entityKey = "http://someurl.com";

        

                final Button btn = (Button) findViewById(R.id.btnCommentView);

                

                btn.setOnClickListener(new OnClickListener() {

                        public void onClick(View v) {                                SocializeUI.getInstance().setSocializeCredentials(consumerKey, consumerSecret);

SocializeUI.getInstance().showCommentView(SampleActivity.this, entityKey);

                        }

                });

        }

}
